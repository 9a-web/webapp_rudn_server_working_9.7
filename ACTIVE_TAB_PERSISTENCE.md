# Сохранение выбора раздела при обновлении страницы

## Описание

При обновлении страницы (F5 или перезагрузке) пользователь остается на том же разделе, на котором находился до обновления.

## Реализация

### Технические детали

**Файл**: `/app/frontend/src/App.js`

#### 1. Инициализация состояния с чтением из localStorage

```javascript
// Состояние для нижнего меню навигации (загружаем из localStorage)
const [activeTab, setActiveTab] = useState(() => {
  const savedTab = localStorage.getItem('activeTab');
  return savedTab || 'home';
});
```

При инициализации компонента:
- Читаем значение `activeTab` из `localStorage`
- Если значение есть - используем его
- Если значения нет (первый визит) - используем дефолтное значение `'home'`

#### 2. Сохранение при изменении раздела

```javascript
// Сохраняем activeTab в localStorage при изменении
useEffect(() => {
  localStorage.setItem('activeTab', activeTab);
}, [activeTab]);
```

При изменении `activeTab`:
- Автоматически сохраняем новое значение в `localStorage`
- Эффект срабатывает при каждом изменении раздела

## Доступные разделы

- `'home'` - Главная (расписание)
- `'tasks'` - Список дел
- `'journal'` - Журнал

## Пользовательский сценарий

1. Пользователь переходит в раздел "Список дел"
2. Значение `activeTab = 'tasks'` сохраняется в `localStorage`
3. Пользователь обновляет страницу (F5)
4. При загрузке приложение читает `localStorage` и устанавливает `activeTab = 'tasks'`
5. Пользователь видит раздел "Список дел", а не главную страницу

## Преимущества

✅ **Непрерывность работы** - пользователь не теряет контекст при случайном обновлении  
✅ **Удобство** - не нужно каждый раз переключаться на нужный раздел  
✅ **Простая реализация** - использует стандартный Web Storage API  
✅ **Без дополнительных зависимостей** - работает на чистом JavaScript
